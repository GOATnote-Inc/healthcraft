coverage. This document defines the coverage gap, stratification axes,
prioritization method, authoring process, and rollout phases.

| Pattern | Description | Current exposure |
|---------|-------------|-----------------|

### 5. Rubric channel fitness

| Channel | Description | Target for new tasks |
|---------|-------------|---------------------|
| world_state | Deterministic audit-log verification | >= 65% of criteria |
| llm_judge | LLM evaluates trajectory | <= 30% of criteria |
| pattern | Regex on agent output | <= 5% of criteria |

New tasks should target 65% world_state criteria from authoring. The v9
deterministic overlay makes this achievable for retrieval, ordering, negation,
and sequencing assertions.

## Prioritization Method

Each uncovered condition is scored:

```
priority = clinical_prevalence * pattern_exposure * rubric_channel_fit
```

Where:
- `clinical_prevalence`: OpenEM `prevalence_rank` (1-5 scale, 5 = most common)
- `pattern_exposure`: How many failure patterns the condition naturally tests
  (1-7 count, normalized to 0-1)
- `rubric_channel_fit`: Fraction of likely criteria that can be world_state
  verified (0-1)

High-prevalence conditions with strong pattern exposure and deterministic
verifiability are authored first.

## Authoring Process

### Per-task workflow

1. **Select condition** from prioritized gap list
2. **Draft task YAML** using `configs/schemas/task.schema.json`
3. **Target 65% world_state criteria** -- use `_QUALIFIER_PARAM_MAP` extensions
   and BEFORE/AFTER operators (v9 channel) from the start
4. **Clinical reviewer sign-off** -- medical accuracy, realistic scenario,
   appropriate difficulty level
5. **Quality gates:**
   - `make preflight` (schema validation)
   - `pytest tests/test_evaluator_integrity/test_task_satisfiability.py`
     (all world_state criteria producible)
   - `scripts/oracle_batch.py --tasks <new-task-id>` (oracle pass)
   - Manual clinical review (physician sign-off)
6. **Add to task inventory** -- file in appropriate category directory

### Naming convention

New tasks continue existing patterns:
- `configs/tasks/<category>/task_<NNN>_<slug>.yaml`
- Task IDs: `<PREFIX>-<NNN>` (CR, CC, IR, MW, SCJ, TR)
- Criteria IDs: `<TASK_ID>-C<NN>`

### Quality bar

Every new task must:
- Map to exactly one OpenEM condition (via `metadata.openem_condition`)
- Have 8-15 binary criteria
- Include at least 1 safety-critical criterion
- Pass schema validation
- Pass satisfiability check (all world_state criteria are producible)
- Pass oracle (a hand-crafted trajectory satisfies all criteria)
- Receive clinical reviewer approval

## Rollout Phases

### Phase A: v0.2 (+100 tasks, target 230/370 = 62%)

**Timeline:** After v0.2 hardening (Phases 0-6) complete.

**Focus:**
- Fill top-priority gaps (high prevalence, strong pattern exposure)
- Add pediatric and OB/GYN coverage
- Add Level 1-2 calibration tasks
- All tasks authored with v9 rubric channel from start

**Deliverable:** 295-task set. Reported as a separate benchmark alongside
the v1 195-task set. Not a replacement.

### Phase B: v0.3 (+100 tasks, target 330/370 = 89%)

**Focus:**
- Environmental, toxicology, psychiatric conditions
- Procedural tasks (intubation, chest tube, LP, central line)
- Cross-condition multi-patient scenarios
- Dynamic-state tasks (requires Phase 3 infrastructure)

**Deliverable:** 395-task set.

### Phase C: v0.4 (+40 tasks, target 370/370 = 100%)

**Focus:**
- Remaining rare conditions
- Cross-cutting multi-condition tasks
- Adversarial tasks (deliberately misleading presentations)

**Deliverable:** 435-task set (370 condition-mapped + ~65 cross-cutting).

## Interaction with Paper

The v1 "195-task set" remains the arXiv v1 benchmark. New task sets are
reported as separate benchmarks with their own pilot results:

- "295-task set (v0.2)" in arXiv v2 addendum
- "395-task set (v0.3)" in separate publication or v3

No v1 number is re-reported with new tasks. The 195-task set comparison
table is frozen.
